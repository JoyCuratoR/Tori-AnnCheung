Here’s what I began with:

Installing Python and Spyder IDE using Miniconda3

What is miniconda? If you Google it, you’ll most likely find Anaconda — miniconda’s mother. Miniconda is the lite version of Anaconda.

If you want to use Python mainly for data science, I recommend you to download miniconda instead of Anaconda. This is because Anaconda is bloated with many unnecessary packages not related to data science.

With a 2 GB disk storage requirement, installing Anaconda means downloading well over 75,000 files. But it’s worth it if you have enough storage space, you’re using Python for multiple reasons, and you want deal with less configuration.

Installation process

Please note: my system is Windows so this tutorial is for this type of OS. Also, there are links here that will re-direct you out of the site but I am not affiliated with any.

Go to conda.io to install miniconda with your preferred Python version. I chose Python 3.10.

Downloading this will install the miniconda3 executable file. Go to path of where the executable is, for me it is: “D:\App Downloads\Miniconda3-py310_22.11.1–1-Windows-x86_64.exe”

1. Click on the executable.

2. Click next.

3. Click to agree to the terms and conditions.

4. Decide if you want to install for all the users on your computer or just yourself (if you’re using a personal computer, it’s suggested to click install just for yourself).

5. Choose a destination folder of where to install all the miniconda3 files.

6. If you haven’t download miniconda or Python on your current computer before, choose the option to add miniconda to your PATH variable (this makes configuration a bit easier).

7. And then check the other recommended boxes.

8. Click install. Don’t worry if it takes a bit of time to install.

Once miniconda is downloaded, open the Anaconda Prompt — it looks exactly like the Command Prompt.

Notice the ‘(base)’ tag coming before the path to your C: drive. The ‘(base)’ indicates you’re in the pre-installed virtual environment. Let’s deactivate this environment by using the command ‘conda deactivate’.

Change directory to your miniconda folder, and navigate to ‘env’ folder. For me my path looks like this: “D:\Py\miniconda3\envs”.

A tip: to change drives just type the letter of your drive with a colon after, like this — ‘d:’

Create a custom virtual environment for data science

Why create a virtual environment? A virtual environment is best described as a programming sandbox, an isolated local workspace that doesn’t affect anything else outside.

Environments prevent any circular dependencies, importing errors, or module errors.

To create one, begin with writing this command in your Anaconda Prompt: ‘conda create -n ds_env’ (you can replace ds_env with what you want to call your environment).

Then we’ll activate the environment with: ‘conda activate ds_env’

Add the conda-forge channel

The conda-forge channel is the best, up-to-date channel to install packages because it’s well-kept by multiple contributors. I highly recommend you install all of your packages from this channel, if possible.

In the Anaconda Prompt, write this command: ‘conda config — env — add channels conda-forge’

The ‘ — env’ option ensures that conda-forge is added only to this environment.

Check that conda-forge has been installed: ‘conda config — show channels’

Now let’s make sure that whenever we install packages, we install them from conda-forge as our default.

‘conda config — env — set channel_priority strict’

Double-check it’s been set: ‘conda config — show channel_priority’

Installing data science packages into the environment

‘conda install numpy pandas scikit-learn matplotlib seaborn’

Double check these packages are installed by running: ‘conda list’

And that’s it!

I am using Spyder IDE for Python because it’s the most similar to R Studio (Posit)’s IDE layout. You can download Spyder IDE using miniconda3 but it’s advisable by them to use their official IDE installer instead.
